Why use md-fetch?

Designed specifically to solve the hardest web scraping challenges for AI applications.

Bypass Anti-Scraping

Uses real headless Chrome and Firefox to bypass 403 Forbidden errors, CAPTCHAs, and execute JavaScript reliably.

Smart HTML Cleaning

Automatically strips unwanted scripts, CSS, ads, and irrelevant metadata while preserving core article structures.

AI & Token Optimized

Outputs lightweight Markdown, drastically reducing token usage and improving context retention for Large Language Models.

Powerful Server Mode

Need to process multiple URLs at once? Start md-fetch serve to spin up a high-performance REST API. Perform batch requests, process pages in parallel, and integrate seamlessly into your AI pipelines.

  • Concurrent batch processing
  • Choose between Chrome, Firefox, or Curl per request
  • Standard OpenAPI 3.0 specification built-in
Explore the API in Swagger

curl
curl -X POST http://localhost:8080/fetch \
  -H "Content-Type: application/json" \
  -d '{
    "urls": [
      "https://example.com/article-1",
      "https://example.com/article-2"
    ],
    "browser": "chrome"
  }'
